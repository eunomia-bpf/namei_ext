\section{实现状态}

当前实现保留一个 kernel-facing BPF ABI，并在 \code{bpf/policies/} 下实现 policy。policy
不是 YAML、JSON 或 DSL。辅助 policy 包括 \file{pass_only.bpf.c}、
\file{redirect_alias.bpf.c} 和 \file{table_redirect.bpf.c}。主线 policy family 包括
\file{build_graph_view.bpf.c}、\file{sandbox_fixture_view.bpf.c}、
\file{checkpoint_restore_view.bpf.c} 和 \file{cache_locality_view.bpf.c}。

当前 ABI 已追加 parent-aware ctx 字段，包括 parent dev、parent ino、parent generation
和 parent flags。这些字段位于 \code{redirect_name} 之后，保持旧字段 offset 不变；
verifier 只允许 BPF 读取这些字段。map-backed policy key 当前使用 event、cgroup id、
parent dev/ino 和 component name。我们暂不把 parent generation 放进 map key，因为
userspace loader 需要用 guest 内 \code{stat()} 构造同一个 key，而普通 \code{stat()}
没有可移植 inode generation 字段。

当前 artifact evidence 分为 KVM gate 和 host-side witness/analysis gate：
\begin{itemize}[leftmargin=*]
\item policy load gate 逐个 load、attach 和 detach policy object。
\item policy semantic gate 验证四类主线 family 的 POC lookup/readdir、
negative pass-through 和 detach 行为。
\item W1--W4 path oracle gate 在 guest 中分别运行 programmable policy 与
table baseline，覆盖 build graph、sandbox fixture、checkpoint/restore 和
cache locality witness entries。
\item W2 nginx real gate 运行真实 nginx binary 的 config parser、worker start、
redirected endpoint HTTP health、worker quit 和 attach 期间 fixture content probes。
该 oracle 只声称 nginx 观察到 config/endpoint alias；cert、secret 和 poison alias
由同一 attach window 内的 direct VFS \code{open/read} probes 覆盖。
\item W3 Redis checkpoint replay gate 运行真实 \code{redis-server} RDB load path，
通过 \file{checkpoint_restore_view.bpf.c} 把可见 \file{dump.rdb} 解析到 hidden
checkpoint backing；W3 Redis macrobench gate 进一步记录 policy/materialized/FUSE
setup/update release input，并明确记录它不是 Podman/CRIU restore。
\item W4 cache content gate 读取 W4 oracle TSV，填充 \file{cache\_rules} map，并验证
verified-hit、stale-fallback、corrupt-reject 和 miss-canonical 分支的内容选择。
\item W4 real ccache gate 在 guest 中运行真实 \code{ccache} cold/hot 编译，记录
Redis/nginx object hash 和 ccache miss/hit 统计，再把 hot object 作为
backing 输入 cache-locality content oracle。
\item W4 real ccache trace gate 在 guest 中用 \code{strace -f -e trace=\%file}
采集真实 ccache hot compile 的 file operations，要求两个 trace 都实际触碰
\code{CCACHE\_DIR}，并明确记录 \code{policy\_executed=false}。
\item W4 trace-derived ccache policy bridge 从真实 ccache strace logs 中抽取成功读取的
cache object component，生成 TSV，并在 guest 中 attach
\file{cache\_locality\_view.bpf.c} 跑 verified-hit content oracle；它明确记录真实
ccache compile 阶段 policy 未执行。
\item W4 real ccache policy-attached compile gate 复制真实 \code{CCACHE\_DIR}，把
trace-derived cache objects 改成 hidden backing，在 attach
\file{cache\_locality\_view.bpf.c} 后运行 Redis/nginx 真实 \code{ccache gcc -c}
hot compile，并验证 output object hash 与 baseline hot object 一致。
\item W4 parent-scoped ccache compile gate 复用同一个 \file{cache\_rules} map，
用 \code{name\_len=0} 的 parent wildcard key 表示一个 ccache leaf parent 下的
verified-hit 对象集合；policy 只对 32 字节 ccache object-shape component 追加
\file{.local}，并要求 component 名称匹配 trace-derived bounded witness。该 gate
在同一 attach window 中验证 \file{metadata.txt} sibling 和一个合法 ccache object
形状但不在 witness 集合中的 sibling 都仍为 PASS；该合法形状 sibling 的
\file{.local} backing 在 attach 前被显式检查为不存在。
\item W4 table-only ccache compile comparator 复用同一份真实 \code{CCACHE\_DIR}、
同一组 Redis/nginx source file 和同一组 trace-derived cache objects，把 policy 换成
\file{table\_redirect.bpf.c} exact redirects；该 comparator 通过当前 sampled witness，
因此作为 C8 的负面证据保留。
\item W4 release counterfactual accounting gate 在 KVM guest 中读取同一 RUN 的真实
ccache trace、parent-scoped policy witness 和 table-only comparator，写出输入哈希与
raw accounting row；当前 row 显示 eligible object-set coverage 为 1，但 sampled
cache-path operation 覆盖率仅为 0.1，table-only 仍通过，因此继续标记 C8 不合格。
\item W1 host build-output witness 校验 Redis/nginx 真实 host source build 的 binary
hash，并明确记录 policy 未执行、KVM 未验证、C8 不合格。
\item W1 KVM policy preprocessing replay witness 在 guest 中 attach
\file{build\_graph\_view.bpf.c}，对真实 Redis/nginx source file 做 preprocessing
replay，并与未 attach policy 的 baseline output 做 byte-for-byte 比较；该 witness
只覆盖 preprocessing 等价性。
\item W1 KVM policy release-binary replay witness 在 guest 中 attach
\file{build\_graph\_view.bpf.c}，对真实 Redis/nginx source tree 运行完整 rebuild，
保存并规范化 debug/build-id 元数据后的 release binary，再与未 attach policy 的
baseline binary 做 byte-for-byte 比较；该 witness 仍显式标记 C8 不合格。
\item table-budget gate 是 host-side analysis target；它只消费 KVM raw JSONL/TSV，
生成 table-only budget accounting，并强制记录 \code{qualified_for_c8=false}。
\end{itemize}
具备 KVM attach 的 gates 证明 policy object 可以沿真实 attach path 影响 lookup、
open/read 和 readdir；host witness、trace-only gate 和 host-side analysis gate 只提供
provenance、candidate evidence 或 accounting。它们都还不是完整 workload output oracle。

KVM policy preprocessing replay witness、KVM policy release-binary replay witness 和 KVM branch
probes。Redis
\code{7.2.14} 和 nginx \code{1.26.3} 的源码 tarball 已固定 SHA256，并由 Makefile
下载、校验、构建和 strace。当前 run id 是
\file{20260615T-parent-key-poc}，生成 source-build manifest、
trace-witness manifest 和 \file{w1-build-output-oracle.jsonl}。Redis trace 有
\code{1,224,839} 行 file-operation trace；nginx trace 有 \code{1,796,476} 行。
trace-witness manifest 只记录手工 candidate 与真实 trace 命中，并用 release-gate
字段和 host-only 执行依据防止后续 report 把它当成 KVM policy oracle。host
build-output oracle 校验两次真实构建产出的 binary hash，但 policy 未执行且 KVM 未验证。
新增 W1 KVM path oracle 只验证这些 candidate entries 在独立
synthetic 目录中的 lookup/readdir path semantics，并把
\file{w1-build-output-oracle.jsonl} 纳入 \file{w1-oracle-inputs.sha256}；所有 W1
row 仍显式标记 \code{qualified_for_c8=false}。新增 W1 KVM policy preprocessing
replay witness 在 guest 中 attach \file{build_graph_view.bpf.c}，对真实 Redis
\file{src/server.c} 和 nginx \file{src/core/nginx.c} 做 preprocessing，并证明 policy
路径产物与 baseline preprocessing 产物相同。2026-06-15 的 parent-aware ABI PoC
进一步让该 replay 使用 guest runtime parent directory 的 \code{dev/ino} 填充 map key，
不再退化为 component-only key。随后新增 W1 KVM policy release-binary replay witness
在 guest 中对 Redis/nginx 真实 source tree 做完整 rebuild，并证明规范化后的 release
binary hash 与 baseline 相同。新增 branch-probe gate 还在 KVM guest 中验证
\code{private.h -> poison.dep} 和 \code{missing.h -> PASS/ENOENT} 两个 build-graph
