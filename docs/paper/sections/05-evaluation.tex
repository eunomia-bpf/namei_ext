\subsection{评估问题}

最终发布级评估将回答五个问题；当前本文主结论只保留证据支持的功能、W2
materialization slice 和 tool-redirect latency slice。Make-owned artifact-discipline
evidence 只说明当前证据可复查；C7 audit 已能生成 sanitized evidence package，但在
clean-checkout reproduction 通过前仍不是 release-level
reproducibility claim。没有达到 \code{qualified\_for\_c8}
的 row 只能说明机制可运行，或者作为反事实负结果，不能进入主结论。
\begin{enumerate}[leftmargin=*]
\item 同一个 kernel ABI 是否能承载四类语义不同的 eBPF path-resolution policy family。
\item lookup、open、access、exec 和 readdir 是否满足每个 use case 的 oracle。
\item 相比 copy tree、symlink forest、bind fanout、FUSE 和 native tool baseline，\namei
是否降低 setup/materialization 成本并保持可接受的 metadata tail latency。
\item \code{table_redirect.bpf.c} 是否在同等 table/update budget 下无法同时满足语义
oracle 和成本门槛。
\item artifact 是否能在干净 checkout 中通过 Makefile-owned Docker/KVM workflow 复现。
\end{enumerate}

\subsection{Baselines 和 gates}

所有 baseline 必须通过 feature-equivalence gate。主 baseline 包括
\code{native-direct}、\code{copy-tree-cp}、\code{symlink-forest-ln}、
\code{bind-fanout-mount}、\code{overlayfs-kernel}、\code{fuse-redirectfs}、
workload-specific native tools，以及 \code{table_redirect.bpf.c}。unsupported baseline
operation 是 hard failure，不能静默跳过。

发布级 run 必须在 KVM guest 中产生。每个 row 必须保存 kernel image hash、kernel config、
Docker image identity、policy source/object hash、verifier log、dmesg、stdout/stderr、
raw JSONL、checker output、workload source hash、alias manifest hash 和 table-only
counterfactual output。若 checker failure、unexpected errno、dmesg warning/oops/panic、
manifest 缺字段或 baseline unsupported operation 出现，相关 cell 失败。

\subsection{Claim gates}

投稿级 evaluation 的主图表必须覆盖 correctness、setup/materialization cost、metadata tail
latency、programmability ablation、scale/failure semantics 和 artifact reproducibility。
当前稿件尚未满足这些图表条件，所以表~\ref{tab:claim-ledger-short} 的 supported slice

不能写成发布级结论，slice-level supported claim 也必须保留命名范围。
每个 subsection 都有降级规则。若四类 family 中少于四类
达到 \code{qualified_for_c8}，论文只能声称已测若干 path-resolution extensions。若

的表述。若 p99 latency 或 setup 成本未达到阈值，论文只能保留 expressiveness/correctness
claim。
