\begin{abstract}
Agent workspaces and environment/cache systems increasingly run unmodified
build, test, and agent tools across short-lived branches, checkpoints, cache
epochs, and validation states. In these transitions, a pathname that resolves
to the wrong existing object can validate the wrong revision, leak writes into
a parent workspace, or feed stale cache inputs into a build. Existing solutions
can present alternate views by materializing namespaces, running filesystem
services, or adopting broader filesystem mechanisms, but those choices also
move ownership of more filesystem behavior than the transition may require.
This proposal separates classification from mechanism. It first classifies
source-system transitions by filesystem effect, and \namei then uses one
decision function for lookup and directory-enumeration events to implement only
pathname-to-object selection and visibility. The lower filesystem retains data,
writes, permissions, persistence, and consistency. \namei is a policy-only VFS
name-resolution extension that does not own filesystem methods, VFS objects,
daemons, or namespace trees. The planned
evaluation separates boundary validity from practical value by requiring two
source-derived KVM transitions to pass correctness and lower-filesystem checks
for the boundary claim before same-oracle baselines are interpreted for
setup/update/runtime cost claims.
\end{abstract}
