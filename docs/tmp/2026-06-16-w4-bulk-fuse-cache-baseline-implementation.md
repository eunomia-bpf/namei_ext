# W4 bulk FUSE cache baseline implementation

> 2026-06-29 baseline scope update: this historical record preserves prior reasoning and results. Current C8/B12 guidance is claim-driven baseline selection; exact-map diagnostics are optional boundary evidence only when precomputed mapping is the competing claim.

Date: 2026-06-16

## Motivation

W4 bulk ccache already has a 20-source/40-object trace shape, a trace-derived
policy bridge, a materialized external baseline, and a policy-attached compile
smoke witness. Before this step, the remaining C2/C8 blocker still included a
FUSE/cache-remap baseline for the same bulk shape. This step adds a Make-owned
KVM target that runs a FUSE `fuse_redirect` cache-view baseline over the same bulk
trace-derived cache objects.

This is an external baseline input, not a positive C2 or C8 result. It must
record `policy_executed=false`, keep `c2_supported=false`, and preserve raw
setup/update/correctness rows.

## Code paths inspected

- `tests/w1_oracle/namei_ext_w1_oracle.c`
  - W1/W2 FUSE helpers and teardown paths.
  - W4 materialized cache baseline row format and correctness checks.
  - W4 bulk ccache source/entry shape generated by the policy bridge.
- `mk/kvm.mk`
  - W4 bulk trace, policy bridge, materialized baseline, and policy compile
    targets.
- `Makefile`
  - Top-level phony target and `make help` entries.

## Design

The target reuses the bulk policy bridge entries TSV, so the baseline covers
the same 20 Redis/nginx source trace shape and 40 trace-derived cache objects.
The runner creates one FUSE mount for the sample cache root. Inside the backing
tree, cache objects are stored as `<object>.local`; the FUSE view maps visible
`<object>` lookups to those backing files and hides the `.local` names from
lookup and readdir.

This design avoids per-leaf FUSE mounts and keeps the baseline close to a
path-remapping cache view. It currently validates setup, visible-content
lookup, readdir visibility, hidden-backing absence, and an update to an existing
backing object. It deliberately does not claim to be a native ccache compile
baseline, because the FUSE operations are read-oriented and do not yet implement
the full write/create surface that ccache may exercise during a real compile.

## Implementation

- Added a W4-specific FUSE root view in
  `tests/w1_oracle/namei_ext_w1_oracle.c`.
- Added `--ccache-fuse-baseline-macrobench`, which emits:
  - `w4-ccache-fuse-baseline-setup`
  - `w4-ccache-fuse-baseline-update`
  - `w4-ccache-fuse-baseline-correctness`
  - `w4-ccache-fuse-baseline-summary`
- Added `kvm-w4-ccache-bulk-fuse-baseline-macrobench` and
  `__phase1_guest_w4_ccache_bulk_fuse_baseline_macrobench` to `mk/kvm.mk`.
- Added the target to top-level `.PHONY` and `make help`.

The guest target fails if setup/update/correctness row counts do not match the
requested sample count, if setup rows do not record one FUSE mount per sample,
if the summary is not pass/feature-equivalent/policy-free, or if the dmesg scan
finds BUG/WARNING/Oops/panic/hung-task evidence.

2. Check target discoverability with `make help`.

   `make kvm-w4-ccache-bulk-fuse-baseline-macrobench

  found the new top-level target.

  `20260616T-w4-ccache-bulk-fuse-baseline-smoke-v1` passed. Its summary row
  recorded one setup row, one update row, one correctness row, one FUSE mount,
  `policy_executed=false`, `feature_equivalent_baseline=true`, `pass=true`,

## Remaining risks

- This is a FUSE cache-view baseline, not a real ccache compile-through-FUSE
  baseline. A future native/FUSE ccache compile baseline needs create/write,
  rename, unlink, chmod/chown, and timestamp operations in the FUSE layer, plus
  output-hash comparison against the policy-attached compile witness.
- A release comparison still needs ledger integration with the same bulk
  proposed-system policy compile shape. This target closes the same-shape FUSE
  cache-view baseline input gap, but it does not establish C2 or C8 support.
- W4 still needs cache-remap/native ccache or BuildKit baselines, a real
  compile-through-FUSE baseline if FUSE is to represent a full ccache tool
  alternative, stale/corrupt/update-window coverage, and table-only budget
  failure evidence.
