## Research Question

RQ exactly as written in the paper:

- RQ1: Can a narrow VFS name-resolution extension express real
  state-dependent path-view policies without taking over filesystem semantics?

Specific uncertainty tested here:

Can the AgentFS-derived workspace path-view slice run through the real
`cgroup/namei_ext` KVM attach path, pass the same source-shaped oracle as a
feature-equivalent FUSE policy, preserve lower-filesystem behavior, and produce
raw operation-weighted evidence strong enough to continue toward final
Experiment A?

Why the answer matters:

Agent workspace lifecycle is the headline experiment for the paper. If it
passes with sufficient source-oracle strength, it supports the central
`sched_ext`-style VFS extension-point claim. If it fails because the source
oracle needs synthetic contents, custom metadata persistence, data-path
mediation, write conflict resolution, or agent orchestration in the filesystem
boundary, it bounds the abstraction.
