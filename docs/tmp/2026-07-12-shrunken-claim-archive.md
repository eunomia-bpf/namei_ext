# Shrunken Claim Archive

Date: 2026-07-12

## Purpose

This note archives the July 10 writing/idea trajectory that turned the project
into a proposal-style, two-transition boundary check. The files are kept as
provenance, not deleted, because they record useful negative evidence and
writing decisions. They no longer define the current idea, RQs, workload route,
or novelty claim.

## Superseded Route

The superseded route centered on:

- table/exact-map impossibility as the novelty gate;
- two selected upstream transitions as the main scientific scope;
- "both transitions must pass" as the boundary-validity condition;
- proposal-only language that treated characterization as future work.

This was too small for the current project. It correctly avoided exclusivity
claims, but it overcorrected by making the paper stop claiming that dynamic,
state-dependent path views are a meaningful workload property.

## Current Route

The current route is:

- `namei_ext` is a `sched_ext`-style VFS extension point;
- the paper positions it between bind/Overlay/materialization and FUSE/custom
  filesystems;
- source-system characterization establishes recurring state-dependent
  path-view transitions;
- AgentFS-derived workspace lifecycle and SWE-Factory-Gym or MEnvData-SWE
  environment/cache transitions are representative KVM deep dives;
- exact-map/table rows are boundary diagnostics only.
